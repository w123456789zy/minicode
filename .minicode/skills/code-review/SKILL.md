---

# Code Review 工作流

1. 列出本次改动涉及的文件（`git diff --name-only HEAD`）
2. 对每个文件：
   - 先看公共 API 签名是否合理
   - 再看边界条件（空值、None、空集合、并发）
   - 再看性能（O(n²)、不必要的 I/O、N+1）
   - 最后看安全（注入、未捕获的异常、敏感信息泄漏）
3. 按"必须改 / 建议改 / 可选"三档输出

# 输出格式

```
必须改：
- ...

建议改：
- ...

可选：
- ...
```
